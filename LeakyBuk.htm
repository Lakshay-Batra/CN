Leaky Bucket Algorithm

The Leaky Bucket Algorithm

        • The Leaky Bucket Algorithm used to control rate in a network.

        • It is implemented as a single-­‐server queue with constant service -me.

        • If the bucket (buffer) overflows then packets are discarded.

                                   

 

(a) A leaky bucket with water.

(b) a leaky bucket with packets.

• The leaky bucket enforces a constant output rate (average rate) regardless of the burst-ness of the input. Does nothing when input is idle.

• The host injects one packet per clock  -ck onto the network. This results in a uniform flow of packets, smoothing out bursts and reducing conges-on.

• When packets have the same size (as in ATM cells), one packet per –ck is okay. For variable length packets though, it is better to allow a fixed number of bytes per -ck. Policing

Ø      Network monitors traffic flows continuously to ensure they meet their traffic contract.

Ø      The process of monitoring and enforcing the traffic flow is called policing.

Ø      When a packet violates the contract, network can discard or tag the packet giving it lower priority

Ø      If congestion occurs, tagged packets are discarded first

Ø      Leaky Bucket Algorithm is the most commonly used policing mechanism

o Bucket has specified leak rate for average contracted rate

o Bucket has specified depth to accommodate variations in arrival rate

o Arriving packet is conforming if it does not result in overflow

Leaky Bucket algorithm can be used to police arrival rate of a packet stream

 Leaky Bucket Algorithm

Ø      The above figure shows the leaky bucket algorithm that can be used to police the traffic flow.

Ø      At the arrival of the first packet, the content of the bucket is set to zero and the last conforming time (LCT) is set to the arrival time of the first packet.

Ø      The depth of the bucket is L+I, where l depends on the traffic burstiness.

                                

                                                                                    Leaky Bucket algorithm used for policing

Ø      At the arrival of the kth packet, the auxiliary variable X’ records the difference between the bucket content at the arrival of the last conforming packet and the interarrival time between the last conforming packet and the kth packet.

Ø      If the auxiliary variable is greater than L, the packet is considered as nonconforming, otherwise the packet is conforming. The bucket content and the arrival time of the packet are then updated.

 

Leaky Bucket Example: - The operation of the leaky bucket algorithm is illustrated in the below figure.

Ø      Here the value I is four packet times, and the value of L is 6 packet times.

Ø      The arrival of the first packet increases the bucket content by four (packet times).

Ø      At the second arrival the content has decreased to three, but four more are added to the bucket resulting in total of seven.

Ø      The fifth packet is declared as nonconforming since it would increase the content to 11, which would exceed L+I (10).

Ø      Packets 7, 8, 9 and 10 arrive back to back after the bucket becomes empty. Packets 7, 8 and 9 are conforming, and the last one is nonconforming.

Ø      Non-conforming packets not allowed into bucket & hence not included in calculations.
